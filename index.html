使用文档 · 知识库

内网穿透是什么？三分钟看懂并上手

内网穿透（Intranet Penetration），又称内网映射、NAT 穿透、端口映射，是在没有公网 IP 的情况下，把本地内网服务安全发布到公网访问的技术。下面先厘清几个关键名词，再用四步带你完成第一条隧道，并对比 frp、nps、ngrok、花生壳等常见方案。

核心名词

Intranet Penetration

内网穿透

通过公网中转节点，把内网设备端口映射到公网地址，让外网可以直接访问家里或公司的内网服务。

NAT Traversal

NAT 穿透

家宽多为 NAT 内网 IP，没有独立公网 IP。穿透技术借助中转服务器绕过 NAT 限制，建立可从外部访问的通道。

Port Forwarding

端口映射 / 端口转发

把公网节点的某个端口，转发到内网设备的指定端口（如 RDP 3389、SSH 22），实现远程直连。

frpc Core

frp 内核

frp 是主流开源内网穿透工具，穿云兼容其 frpc 客户端内核，并在其上封装了守护、调度与图形控制台。

常见问题

关于内网穿透，你可能想问

关于内网穿透、没有公网 IP、frp / nps / ngrok / 花生壳的对比与安全性等高频问题，都在这里。

什么是内网穿透？没有公网 IP 也能用吗？
内网穿透（又叫内网映射、NAT 穿透）是通过位于公网的中转节点，把没有公网 IP 的本地设备端口安全地映射到一个公网地址上，从而在外网直接访问家里或公司的内网服务。穿云依托全国多线节点搭建加密隧道，即使运营商只分配内网 IP、没有公网 IP，也能稳定地把本地服务发布到公网。
穿云和 frp、nps、ngrok、花生壳有什么区别？
穿云兼容 frp 生态，并在其内核之上自研了跨平台守护客户端、智能节点调度与图形化控制台。相比自己用 frp、nps 搭建，无需租服务器、写配置、盯进程；相比 ngrok、花生壳等 SaaS，穿云提供全国多线节点、配置热更新、SSL 自动签发与 NAS 一键接入，中文界面、开箱即用。
穿云支持哪些系统和设备？
客户端覆盖 Windows、Linux、macOS，以及飞牛 fnOS、群晖等主流 NAS，可注册为系统服务开机自启。常见场景包括远程桌面、SSH、内网建站、NAS 远程访问、游戏联机与内部接口联调。
内网穿透安全吗？
穿云隧道全程 TLS 加密，按设备签发最小权限、相互隔离的一次性凭证，账户密码加盐哈希存储，并对远程桌面、SSH 等高危端口做实时巡检告警，从传输、凭证到审计层层加固。
支持哪些协议和端口映射方式？
支持 TCP、UDP、HTTP、HTTPS、STCP 等多种隧道协议，可用于端口转发、内网建站、点对点安全访问等场景；端口自动分配，支持绑定自定义域名与 SSL 证书自动签发。

内网穿透和 VPN 有什么区别？
内网穿透和 VPN 是两类不同的技术。VPN 侧重把设备接入一个远程虚拟局域网、改变整体上网出口；内网穿透只是把某个本地服务（如远程桌面、NAS、自建网站）通过公网中转节点定向映射到公网，让外网能访问它，并不改变你的上网出口。穿云是专注内网穿透的平台，帮你把没有公网 IP 的本地服务安全发布到公网，不是 VPN，也不用于访问境外网站。
没有公网 IP 怎么做内网穿透？
家庭宽带与多数运营商通常只分配内网 IP、没有独立公网 IP，本地服务无法被外网直接访问。穿云通过部署在公网的多线中转节点建立加密隧道，把你本地的端口映射到一个固定公网地址上，因此即使完全没有公网 IP，也能稳定地从外网访问家里的 NAS、远程桌面或自建网站。
群晖 / 飞牛 fnOS 等 NAS 如何实现外网访问？
在 NAS 上安装穿云客户端（支持飞牛 fnOS、群晖、威联通 QNAP 等），登录后设备自动注册；在控制台为 NAS 的管理后台或相册、影音服务创建隧道并选择就近节点，保存后即可用分配的公网地址或自定义域名在任意网络下访问家中 NAS，无需公网 IP，也不必在路由器上做端口映射。
